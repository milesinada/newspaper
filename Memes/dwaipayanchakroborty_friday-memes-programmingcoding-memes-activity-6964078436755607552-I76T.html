        The concept of 'full stack' has become pretty old. The cloud has evolved - we should start calling it 'full cycle' now. Leveraging the cloud, a developer can carry forward even to the 'Develop - Operate - Support' phases of SDLC. But that's not it, obviously. Leveraging cloud platforms come with many other advantages: - lesser overheads that come with managing servers on-prem. - better scalability and security, derived from the tech giant(s). - fast delivery, hence faster iterations. - cost-effectiveness, to name a few. Containerization of applications is a technology, that is growing massively pop, and not to mention widely adopted in the software development field - due to its enormous benefits. This too is extensively cloud-based. Docker is one of the many such toolkits that provisions this. As the name suggests, it packages and 'contains' the environment in which an application is supposed to run. This means the entire runtime environment required for that app to run successfully is made portable in a way that is able to run on any hardware. This takes away the manual, time-consuming steps of setting up the environment all by yourself, which almost always includes dealing with errors. AWS is one of the most pop cloud-based PaaS that provides services that use the principles of containerization, which makes running, deploying, and managing apps at scale easier. So, if you are looking to learn more about the containerization of apps using AWS, and how they can help you in the overall software development process. This course instructed by Russell Sayers & Morgan Willis is a good start to getting familiar with tools like Docker and k8s, and services like App runner, ECS, EKS, and more. #softwaredevelopment #tech #aws #docker #cloud
